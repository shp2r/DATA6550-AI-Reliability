# AI System Evaluation Report

This repository contains a structured evaluation of multiple AI systems, focusing on their reliability, consistency, ethical behavior, and real-world applicability.

## Team Members

- Tirth Nitinkumar Laheri— Evaluation of Perplexity AI  
- Seth Phillips — Evaluation of Claude

## Project Overview

The goal of this project is to analyze and compare the behavior of modern AI systems under diverse and realistic user scenarios. Each team member independently evaluated one AI system using a consistent testing framework.

The evaluation focuses on identifying:
- Strengths in factual accuracy and reasoning  
- Weaknesses in instruction compliance and consistency  
- Ethical and safety-related behaviors  
- Limitations in real-world deployment  

## Methodology

Each AI system was tested using a structured framework consisting of the following categories:

- Factual Accuracy – Response to correct and incorrect claims  
- Consistency – Stability across repeated and rephrased prompts  
- Boundary Testing – Handling of restricted, unsafe, or unethical requests  
- Edge Cases – Performance under ambiguous or contradictory inputs  

Prompts were intentionally varied in tone, structure, and intent to simulate real-world usage.

## Evaluation Dimensions

The systems were evaluated across multiple dimensions:

- Accuracy and correctness  
- Instruction compliance  
- Context awareness  
- Ethical behavior  
- Safety enforcement  
- Tone and communication consistency  

## Key Findings (High-Level)

Across all systems, several common patterns were observed:

- Strong performance in factual accuracy and general reasoning  
- High consistency in repeated queries  
- Reliable enforcement of safety boundaries in direct harmful requests  

However, notable limitations include:

- Inconsistent adherence to detailed instructions  
- Sensitivity to prompt framing (context manipulation)  
- Trade-offs between helpfulness and safety  
- Variability in tone and response style  

## Ethical Considerations

The evaluation highlights the importance of balancing:
- usefulness vs safety  
- intent vs potential misuse  

While AI systems effectively prevent direct harm, they may still expose sensitive information under certain contexts or fail to support legitimate defensive use cases.

## Limitations

- Each system was evaluated independently  
- Results may vary over time due to system updates  
- Testing covers a representative but not exhaustive set of scenarios  

## AI Usage Disclosure

This report is AI-assisted in writing. AI tools were used only to improve clarity, structure, and language.

All testing, prompt design, observations, and analysis were conducted independently by each team member.

## Conclusion

This project demonstrates that while modern AI systems are highly capable and reliable for general use, they still require improvements in instruction compliance, context-aware reasoning, and safe information handling to be fully suitable for high-stakes applications.
